# 開發結案回顧 (Retrospective)

本程序在待辦事項完結後啟動，旨在整理開發過程中的技術收穫，並清理開發狀態。

## 1. 開發筆記與摘要
- **待辦事項執行者狀況回顧**: 讀取進度管理檔案中的歷史標註與開發紀錄。
- **技術問題紀錄**: 總結本次開發中遇到的解決方案、非預期 Bug 以及成功的實作模式。

## 2. 開發知識庫更新
- **高品質模式紀錄**: 將具備複用價值的代碼片段、新組件模式或優化的 SQL 規則寫入 `.agent/knowledge` 對應分類。
- **更新地雷待辦事項清單**: 若遇到特定技術坑位，必須更新 `gotchas.md` 以防止未來重複發生。

## 3. 清理環境與存檔
- **心得存檔**: 將本待辦事項的核心摘要與技術變更點寫入專案日誌。
- **環境清理**: **清空** 進度管理檔案末端的「待辦事項狀態」區塊，確保下一個待辦事項建立有乾淨的開始。

---
> [!IMPORTANT]
> **結案守則**：整理桌面是為了讓下一次開發更順暢。
