# 程式碼審查流程

**[待辦事項執行者]**: 本工作流啟動後，AI 強制載入 **系統架構師** 與 **品質工程師** 視角。

## 1. 待辦事項結案審核
- **分析內容**: 讀取當次開發紀錄與待辦事項清單。
待辦事項管理檔案確認執行背景。
- **目標**: 針對變更檔案進行架構與健壯性雙重檢查。

## 2. 深度分析規範
- **技術棧合規審核**: 必須讀取技術基底，檢核是否使用了非開發計畫中的技術或第三方庫。
- **原生標籤檢核**: 
    - 嚴禁出現原生 HTML 標籤。
    - **解決方案**: 若發現原生標籤，必須要求開發者替換為元件庫對應組件。
- **介面樣式與情感審查**: 必須對應介面樣式規範，檢查圓角、毛玻璃以及陰影擴散是否落位。

## 3. 輸出審查報告
- **結構**: 
    1.  **架構完整性評分**
    2.  **介面美感回報**
    3.  **修訂建議清單**
- **回報**: 在任務清單中標註審查完成。

---
> [!IMPORTANT]
> **審查官守則**：審查的核心不是找錯，而是確保每一行代碼都具備專案定義的技術與美感基因。
