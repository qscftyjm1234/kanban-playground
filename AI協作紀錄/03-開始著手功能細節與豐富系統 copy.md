## 決策與理由

一. 建立登入auth 登入功能
- 考慮到安全與便利性，使用 JWT 做驗證並把 Token 存本地。
- 介面設計成對分式佈局，左側放公司識別 Logo 加強識別度。

二. 看板群組管理
- 為了讓專案分流，開發了多看板架構，並搭配 Zustand 狀態管理實現切換。
- 後端實作連鎖刪除機制，確保移除看板時資料清理完整。

三. 增加table功能
- 考慮到卡片過多時的辨識度，多做了一個列表模式好讓資料密度提高。

四. 增加看板搜尋功能
- 直接在前端用 filter 處理關鍵字過濾，不用透過 API 反應速度最快。
